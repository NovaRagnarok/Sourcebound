## Problem Statement

Sourcebound already has persisted jobs and degraded-mode runtime signals, but day-to-day operation still depends on reading raw status fields or piecing together warnings across the API and UI. This chunk should improve the legibility of partial, retry, and failure states without widening into broader workflow redesign.

## Implementation Plan

1. Inspect the current job service, job API routes, runtime status summaries, UI job surfaces, and existing tests.
2. Identify the smallest missing telemetry or presentation changes that would make partial, retry, failure, and degraded-mode states easier to understand in daily use.
3. Implement bounded changes in the allowed service, API, runtime-status, and UI files only.
4. Add or update tests for at least one end-to-end partial or retry scenario plus the surfaced degraded-mode summaries.
5. Run chunk-local verification and repo-wide verification, then prepare the chunk for review.

## Acceptance Criteria

- Partial, retry, stalled, and failure states are legible in the existing job and runtime-status surfaces.
- At least one end-to-end partial or retry scenario is covered.
- No unrelated files are edited.
