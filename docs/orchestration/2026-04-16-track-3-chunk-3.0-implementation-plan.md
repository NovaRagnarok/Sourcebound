## Problem Statement

The repo already explains the supported runtime boundary, but operators still have to infer the real recovery workflow from scattered notes. This chunk should turn that guidance into an actual runbook with explicit backup, restore, Qdrant rebuild, reseed, upgrade, and rollback steps, while staying inside the existing deployment and operator-doc surfaces unless a small CLI clarification is truly necessary.

## Implementation Plan

1. Inspect the current deployment docs, operator docs, README, and existing CLI recovery commands.
2. Identify the smallest missing procedural guidance for backup, restore, Qdrant rebuild, reseed, upgrade, and rollback.
3. Consolidate those steps into a practical operator runbook in the allowed docs, with exact command examples where the repo already supports them and CLI changes only if a current command or output needs clarification to support the documented workflow.
4. Verify the documented commands against the current repo behavior and keep the runbook aligned with shipped boundaries.
5. Run chunk-local verification for any changed commands or docs-backed checks, then run repo-wide verification if code changed.
6. Prepare the chunk for review once the runbook is explicit and bounded.

## Acceptance Criteria

- Backup targets and restore steps are explicit.
- Qdrant rebuild, reseed, upgrade, and rollback steps are explicit and executable from the documented surface.
- The runbook stays aligned with the currently supported self-host shape.
