## Problem Statement

CI currently proves linting, typing, and the test suite, but it does not explicitly run the default-stack newcomer smoke path. This chunk should add a dedicated smoke gate to the main PR workflow, explicitly boot the default services, and make `docker compose` startup failures easier to distinguish from smoke-script app failures.

## Implementation Plan

1. Inspect the current CI workflow, smoke script, and compose setup.
2. Add the smallest bounded CI job or step set needed to boot Postgres and Qdrant and run the newcomer smoke script in the PR workflow.
3. Make CI output distinguish `docker compose` startup failures from smoke-script app failures, and nothing broader.
4. Keep the workflow scoped to the documented default local stack.
5. Run local verification for the changed workflow-adjacent files, then prepare the chunk for review.

## Acceptance Criteria

- The main PR workflow runs the default-stack smoke path on every pull request.
- The smoke gate boots Postgres and Qdrant before running the smoke script.
- Failure output makes infra problems meaningfully distinguishable from app failures.
- Chunk changes stay within the allowed files.
