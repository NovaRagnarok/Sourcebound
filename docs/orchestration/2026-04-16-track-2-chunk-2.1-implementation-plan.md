## Problem Statement

The repo can already talk to Zotero, but the live-corpus path still feels fragile when configuration is incomplete, write-back is unavailable, or attachment/note discovery is hard to interpret. This chunk should tighten the live diagnostics and intake-path clarity without widening into broader ingestion redesign.

## Implementation Plan

1. Inspect the current Zotero adapter, CLI diagnostics, and live integration tests.
2. Identify the smallest missing diagnostics around configuration, write-path readiness, and document discovery behavior.
3. Add bounded improvements in the adapter and CLI only where the current live path is ambiguous or under-explained.
4. Expand live integration coverage so Zotero configuration failures and attachment/note discovery outcomes are easier to interpret.
5. Run chunk-local and repo-wide verification, then prepare the chunk for review.

## Acceptance Criteria

- `zotero-check` gives concrete fixes for incomplete or invalid setup.
- Attachment and note discovery failures are easier to interpret.
- Write-path readiness is clearer when live Zotero is configured versus unavailable.
- No unrelated files are edited.
