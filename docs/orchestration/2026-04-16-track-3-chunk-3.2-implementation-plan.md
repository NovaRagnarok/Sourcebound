## Problem Statement

Sourcebound now has broader extraction-eval scenarios and suite artifacts, but retrieval quality still is not enforced as a stable gate in automation. This chunk should make the benchmark summary more diffable and surface at least one retrieval regression check in CI or an equivalent release-facing automation path without redesigning the benchmark system.

## Implementation Plan

1. Inspect the current extraction-eval artifact shape, retrieval metadata surfaces, and CI workflow.
2. Identify the smallest changes needed to make benchmark outputs stable and diff-friendly across runs.
3. Add one bounded automation surface for retrieval regression visibility, preferably in existing CI.
4. Update extraction-eval docs so operators know what the gate checks and how to interpret failures.
5. Run chunk-local and repo-wide verification, then prepare the chunk for review.

## Acceptance Criteria
