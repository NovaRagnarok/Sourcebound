## Problem Statement

The extraction benchmark is now reproducible, but it is still narrow: only two checked-in datasets exist and the CLI emphasizes one dataset at a time. That makes progress easy to over-read and hard to compare across scenarios. This chunk should add a few more evaluation scenarios and a comparison-friendly summary surface without turning benchmarking into a broader reporting system.

## Implementation Plan

1. Inspect the current extraction-eval dataset catalog, artifact shape, CLI output, and existing test coverage.
2. Add the smallest bounded set of new checked-in evaluation scenarios needed to bring the catalog beyond the current two presets.
3. Tighten the existing eval summary, report, or CLI surface so operators can compare dataset results more easily over time.
4. Expand extraction-eval tests to pin dataset discovery, summary output, and comparison-friendly reporting.
5. Update the extraction-eval docs so the new scenario breadth and comparison workflow are explicit.
6. Run chunk-local and repo-wide verification, then prepare the chunk for review.

## Acceptance Criteria

- Add 2 to 3 new checked-in extraction-eval datasets.
- The existing extraction-eval CLI or generated artifacts are easier to compare over time than the current single-dataset-only view.
- Extraction-eval docs reflect the broadened scenario set and comparison workflow.
- No unrelated files are edited.
