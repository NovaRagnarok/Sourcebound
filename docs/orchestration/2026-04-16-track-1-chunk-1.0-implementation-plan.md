## Problem Statement

The current baseline gap has two tightly related test-contract problems:

- the Zotero write-path tests exercised mocked writes without explicitly setting `ZOTERO_API_KEY`
- one API integration test assumed missing default services without explicitly stubbing the runtime probes, making it environment-sensitive

## Implementation Plan

1. Confirm the current failing behavior and isolate it to the Chunk 1.0 Zotero write path.
2. Preserve real write safety by keeping production write operations credential-gated.
3. Make the test contract explicit:
   - credentialed live writes must remain gated by `ZOTERO_API_KEY`
   - mocked unit tests must isolate the adapter boundary instead of relying on ambient credentials
   - non-credentialed behavior must be asserted explicitly in tests rather than passing by accident
4. Update targeted tests so both contracts are clear and durable.
5. Keep product behavior unchanged unless a test cannot be made deterministic any other way.
6. Run chunk-scoped verification first, then full local `pytest` as the chunk acceptance gate.
