## Problem Statement

Sourcebound now has bounded deployment and recovery guidance, but release
discipline is still scattered. The repo needs one clear release checklist, one
rollback path, and one issue-triage standard that all stay aligned with the
current trusted-operator support matrix.

## Implementation Plan

1. Inspect the current support-policy, deployment, and recovery wording to find
   the smallest canonical release-operations seam.
2. Add one focused release-operations doc that covers release checklist,
   rollback guidance, and issue-triage expectations for the current product
   boundary.
3. Align README and operator/deployment docs to point at the canonical support
   matrix and the new release-operations doc without duplicating a second
   release policy.
4. Add one GitHub issue template that captures the minimum support-matrix and
   rollback context needed for triage.

## Acceptance Criteria

- a release checklist exists for the current trusted-operator product boundary
- rollback guidance exists and points at the same supported runtime path
- issue triage standards exist and are reflected in a repo issue template
- support-matrix wording stays version-aligned across the touched docs

## Non-Goals

- no new deployment topology, CI release automation, or packaging system
- no auth, routing, or runtime-behavior changes
- no incident-response platform or pager workflow
- no support-policy expansion beyond the current trusted-operator boundary

## Reviewer Notes
