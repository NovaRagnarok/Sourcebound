## Problem Statement

Sourcebound documents a minimum trusted-operator deployment shape, but the repo
does not yet ship one concrete, tested single-host path for an app container
with the in-process worker enabled, plus Postgres, Qdrant, and a reverse proxy.
The current compose file only brings up Postgres and Qdrant, so the docs and
verification story are not yet backed by one real self-host deployment path.

## Implementation Plan

1. Inspect the current runtime entry points, compose setup, and existing
   newcomer/default-stack verification paths.
2. Define the smallest supported single-host deployment layout that includes an
   app container with the in-process worker enabled, plus Postgres, Qdrant, and
   a reverse proxy, without implying broader production hardening.
3. Add or update the deployment assets and docs so the guide matches the real
   deployment shape.
4. Add bounded verification that exercises the supported deployment path and
   keeps health/readiness guidance aligned with the docs.
5. Prepare the chunk for read-only review without widening into public-internet
   hardening, auth implementation, or optional-integration productization.

## Acceptance Criteria

- one deployment guide is real and tested
- health and verification tooling match the guide
- no unrelated runtime or auth scope is introduced
