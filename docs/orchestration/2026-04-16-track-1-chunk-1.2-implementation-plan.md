## Problem Statement

The newcomer smoke path currently assumes a fixed app port, a single server startup shape, and a simple readiness probe. That makes it fragile when a port is already occupied or when the wrong process answers the probe. This chunk should make the smoke path conflict-aware, process-aware, and clearer on cleanup/failure without redesigning the broader startup flow.

## Implementation Plan

1. Inspect the current smoke script for assumptions around port selection, process identity, readiness checks, and cleanup.
2. Add the smallest bounded mechanism needed to avoid or detect environment collisions clearly.
3. Make the script verify that the expected server process is the one answering the HTTP probes.
4. Keep cleanup predictable and leave unrelated host processes untouched.
5. Add or update tests for at least one collision-aware path and one clearer startup-failure path.
6. Run chunk-scoped verification first, then the strongest broader verification for this slice.

## Acceptance Criteria

- The newcomer smoke path's own port handling is configurable or conflict-aware.
- The script validates process identity before or alongside URL probing.
- Environment collisions fail fast with a clear reason.
- Chunk changes stay within the allowed files.
