## Problem Statement

Sourcebound now has shared writer/operator auth, but important trust-boundary
 actions still lack durable actor-aware audit history. This chunk should attach
 actor identity where auth is enabled and make core canon/review history
 queryable, without widening into collaboration workflow or release-process
 work.

## Implementation Plan

1. Inspect the current review and Bible provenance models to find the smallest
   durable audit seam already closest to reality.
2. Add only the actor-aware history fields and persistence needed for core
   canon and review actions.
3. Thread the authenticated actor through the bounded route-to-service seams
   needed for those writes.
4. Surface that history through the existing service/query surfaces needed for
   the chunk acceptance criteria.
5. Verify the bounded audit history with focused service and route tests plus
   the strongest relevant regression checks.

## Acceptance Criteria

- review approve and reject events persist actor identity in `ReviewEvent`
- Bible profile save, section update, regenerate request, and export request
  persist actor identity in `CanonAuditEvent`
- the existing API and service responses for the touched review and Bible
  actions expose that stored actor-aware history
