## Problem Statement

The default retrieval path now depends on Qdrant being healthy, initialized, and accurately described to operators and tests. This chunk should add bounded guard rails around projection freshness, fallback behavior, and retrieval metadata so regressions in the default path are caught before they silently degrade query quality.

## Implementation Plan

1. Inspect the current query service, Qdrant adapter fallback behavior, and existing retrieval tests.
2. Identify the smallest missing guard rails around projection freshness, backend labeling, quality tier, and degraded fallback behavior.
3. Add bounded implementation changes in the query service and adapter only as needed to surface or preserve the expected metadata.
4. Expand query-service coverage so default-path Qdrant behavior and degraded fallback behavior are explicitly pinned.
5. Run chunk-local and repo-wide verification, then prepare the chunk for review.

## Acceptance Criteria

- Retrieval backend and quality tier are verified for the default path.
- Degraded-Qdrant fallback behavior is explicitly tested.
- Query metadata stays accurate when the default retrieval path falls back or degrades.
- No unrelated files are edited.
