## Problem Statement

Sourcebound has the raw writer/operator boundary and Bible surfaces, but it
does not yet present one explicit collaboration loop end to end. This chunk
should choose the Bible handoff loop and harden the docs, permissions cues, UI
handoff messaging, and end-to-end verification around that one workflow only.

## Implementation Plan

1. Inspect the current Bible routes, workspace summary, and UI screens to find
   the clearest existing handoff path between writer and operator.
2. Document the selected Bible handoff loop step by step in one focused
   workflow doc and align any README pointer needed for discoverability.
3. Tighten permission and route messaging only where needed so writer versus
   operator responsibilities are explicit for this loop.
4. Surface the handoff clearly in the workspace UI without redesigning
   unrelated screens.
5. Add one end-to-end integration scenario proving the documented loop works.

## Acceptance Criteria

- the Bible handoff loop is documented step by step from writer setup/editing
  to operator regenerate/export handoff
- permissions and role cues align with the existing writer/operator boundary
  for that loop
- the UI clearly shows the handoff without introducing a second workflow
- one end-to-end scenario proves the loop works

## Non-Goals

- no generic collaboration platform or second workflow
- no review, research, intake, or release-operations redesign
- no new auth provider, session, or role model work
- no deployment, recovery, or CI changes

## Reviewer Notes

Review only the Bible handoff loop. Reject drift into a broader workflow
system, multi-user auth redesign, or unrelated UI cleanup. Approve only if the
selected loop is documented, permission-aligned, surfaced in the UI, and
proven by one end-to-end scenario. Do not treat adjacent workspace actions such
as review, research, ask, or background-job cues as a second workflow for this
chunk.
